The Architecture Die Architektur

Click any component to see what it does and how it connects. Klick auf eine Komponente um zu sehen was sie tut und wie sie verbunden ist.

ClarifyKlären
📥 Capture
Quick note-taking, PARA categorization Schnelle Notizen, PARA-Kategorisierung
Captures ideas and notes into a structured inbox using the PARA method (Projects, Areas, Resources, Archive). Prevents ideas from getting lost during focused work. Erfasst Ideen und Notizen in einer strukturierten Inbox nach der PARA-Methode (Projects, Areas, Resources, Archive). Verhindert dass Ideen bei fokussierter Arbeit verloren gehen.
🔬 Distill
Progressive summarization in 4 layers Progressive Zusammenfassung in 4 Schichten
Layer 1: Full notes. Layer 2: Bold passages. Layer 3: Highlighted core. Layer 4: Executive summary. Each layer compresses by ~70%. Schicht 1: Vollständige Notizen. Schicht 2: Fettgedruckte Passagen. Schicht 3: Markierter Kern. Schicht 4: Executive Summary. Jede Schicht komprimiert um ~70%.
🔍 Analyze
ReAct analysis (Plan-Act-Reflect cycle) ReAct-Analyse (Plan-Act-Reflect-Zyklus)
Structured analysis using Plan (what to examine), Act (gather evidence), Reflect (what does it mean). Prevents premature conclusions by enforcing a thinking cycle. Strukturierte Analyse mit Plan (was untersuchen), Act (Evidenz sammeln), Reflect (was bedeutet das). Verhindert voreilige Schlüsse durch einen erzwungenen Denkzyklus.
BuildMachen
✍️ Express
Output generation with POPP method + UX principles Output-Generierung mit POPP-Methode + UX-Prinzipien
POPP = Purpose, Outline, Prose, Polish. Generates texts, posts, documents with consistent quality. Applies UX writing principles: clear, concise, useful. POPP = Purpose, Outline, Prose, Polish. Generiert Texte, Posts, Dokumente mit konsistenter Qualität. Wendet UX-Writing-Prinzipien an: klar, knapp, nützlich.
VerifyPrüfen
🎯 Signal-Check
4-axis substance analysis 4-Achsen-Substanzanalyse
Checks 4 axes: Meaning (does it say something?), Facts (is it true?), Framing (is the perspective fair?), Context (is anything missing?). Quick check, ~500 tokens. Prüft 4 Achsen: Bedeutung (sagt es etwas?), Fakten (stimmt es?), Framing (ist die Perspektive fair?), Kontext (fehlt etwas?). Quick Check, ~500 Tokens.
⚔️ Challenge
Adversarial stress-testing with 3 attack roles Adversarialer Stress-Test mit 3 Angriffsrollen
Three attack roles: Devil's Advocate (logical weaknesses), Domain Expert (technical gaps), Skeptic (unstated assumptions). Hardens outputs before publication. Drei Angriffsrollen: Devil's Advocate (logische Schwächen), Domain Expert (fachliche Lücken), Skeptiker (unausgesprochene Annahmen). Härtet Outputs vor Veröffentlichung.
🛡️ Quality-Gate
Risk-based triage (levels 0–3), quality scoring Risikobasierte Triage (Stufen 0–3), Quality-Scoring
Level 0: No risk, no check. Level 1: Signal-check + score. Level 2: + external verification. Level 3: Full review + challenge. Automatic escalation based on risk indicators (numbers, claims, causal statements). Stufe 0: Kein Risiko, kein Check. Stufe 1: Signal-check + Score. Stufe 2: + externe Verifikation. Stufe 3: Full Review + Challenge. Automatische Eskalation basierend auf Risiko-Indikatoren (Zahlen, Behauptungen, Kausalaussagen).
Meta
💾 Handoff
Session state persistence for seamless continuation Session-State-Persistenz für nahtlose Fortsetzung
Saves current session state (open tasks, decisions, context) so the next session picks up exactly where you left off. Triggered automatically at 25% context remaining. Speichert aktuellen Session-State (offene Aufgaben, Entscheidungen, Kontext) damit die nächste Session genau dort weitermacht. Wird automatisch bei 25% verbleibendem Kontext ausgelöst.

🧠 The Brain Das Gehirn

Config File (CLAUDE.md) — The AI's operating manual. Defines workflow, routing rules, and quality standards. Config-Datei (CLAUDE.md) — Das Betriebshandbuch der AI. Definiert Workflow, Routing-Regeln und Qualitätsstandards.

What it containsWas sie enthält

3-Phase Workflow: Clarify → Build → Verify. No phase gets skipped.

Cognitive Mode Detection: Automatically recognizes whether you need research, writing, analysis, or code and routes to the right skills.

Routing Table: Maps skill names to phases. The AI knows which tool to use for each task type.

Quality Principles: Traceability, source criticism, honesty about uncertainty. Baked into every response. 3-Phasen-Workflow: Klären → Machen → Prüfen. Keine Phase wird übersprungen.

Kognitive Modus-Erkennung: Erkennt automatisch ob du Recherche, Schreiben, Analyse oder Code brauchst und leitet an die richtigen Skills weiter.

Routing-Tabelle: Bildet Skill-Namen auf Phasen ab. Die AI weiß welches Tool für welchen Aufgabentyp geeignet ist.

Qualitätsprinzipien: Nachvollziehbarkeit, Quellenkritik, Ehrlichkeit bei Unsicherheit. In jede Antwort eingebaut.

Memory Architecture Gedächtnis-Architektur
Short-term Kurzzeit
Current conversation (~200K tokens, 1 session) Aktuelle Konversation (~200K Tokens, 1 Session)
The active conversation window. Everything you say and every file the AI reads lives here. Limited by context window size. When it fills up, handoff saves the state. Das aktive Konversationsfenster. Alles was du sagst und jede Datei die die AI liest lebt hier. Begrenzt durch die Context-Window-Größe. Wenn es voll ist, sichert Handoff den Zustand.
Mid-term Mittelzeit
Project states, routing log (weeks to months) Projekt-States, Routing-Log (Wochen bis Monate)
Project files track open todos, decisions, and progress. The routing log records which skills were used and why. Survives across sessions. Updated during work, not at session end. Projektdateien tracken offene Todos, Entscheidungen und Fortschritt. Das Routing-Log zeichnet auf welche Skills genutzt wurden und warum. Überlebt Sessions. Wird während der Arbeit aktualisiert.
Long-term Langzeit
Config file, knowledge base, glossary (permanent) Config-Datei, Wissensbasis, Glossar (permanent)
The config file itself, Obsidian vaults, SQLite databases, and the memory file. These persist indefinitely and form the AI's long-term understanding of your work and preferences. Die Config-Datei selbst, Obsidian Vaults, SQLite-Datenbanken und die Memory-Datei. Diese bestehen unbegrenzt und bilden das langfristige Verständnis der AI für deine Arbeit und Präferenzen.

Skill Chains Skill-Ketten

Skills don't work in isolation. These feedback loops produce better results than any single skill. Skills arbeiten nicht isoliert. Diese Feedback-Schleifen produzieren bessere Ergebnisse als jeder einzelne Skill.

1. Evaluator-Optimizer 1. Evaluator-Optimierer

express → signal-check → express ↻

Write, check substance, improve. Iterate until quality threshold is met. Schreiben, Substanz prüfen, verbessern. Iterieren bis die Qualitätsschwelle erreicht ist.

2. Evaluator-Challenger 2. Evaluator-Challenger

express → challenge → express ↻

Write, stress-test, harden. Adversarial review strengthens arguments. Schreiben, stressen, härten. Adversariales Review stärkt Argumente.

3. Quality Gate Loop 3. Quality-Gate-Schleife

express → quality-gate → express ↻

Write, verify with external checks, revise. Risk-based triage determines check depth. Schreiben, mit externen Checks verifizieren, überarbeiten. Risikobasierte Triage bestimmt die Prüftiefe.

4. Knowledge Cycle 4. Wissenszyklus

capture → distill → express → signal-check

Capture raw input, distill to essence, express as output, verify quality. The learning pipeline. Rohen Input erfassen, zum Kern destillieren, als Output ausdrücken, Qualität prüfen. Die Lernpipeline.

Context Management Kontext-Management

How sessions chain together without losing state. The hooks handle it automatically. Wie Sessions verkettet werden ohne State zu verlieren. Die Hooks erledigen das automatisch.

📊 context-statusline.js

Shows context usage as a percentage in real-time. You always know how much capacity is left. Zeigt den Kontext-Verbrauch als Prozentanzeige in Echtzeit. Du weißt immer wie viel Kapazität übrig ist.

⚠️ context-monitor.js

Warns at 35% remaining. Triggers handoff at 25%. Prevents mid-task context loss. Warnt bei 35% verbleibend. Löst Handoff bei 25% aus. Verhindert Context-Verlust mitten in der Aufgabe.

🔄 session-start.sh

Reads previous handoff at startup. The new session continues where the last one stopped. Liest vorherigen Handoff beim Start. Die neue Session macht dort weiter wo die letzte aufgehört hat.

git clone https://github.com/janrummel/claude-orchestrator-starter.git
cd claude-orchestrator-starter

# Copy to your Claude config
cp CLAUDE.md.example ~/.claude/CLAUDE.md
cp -r orchestrator/ ~/.claude/orchestrator/
cp -r memory/ ~/.claude/memory/

# Start Claude Code
claude
